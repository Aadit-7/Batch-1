3. Nominculture/ Rules

    a. Variable/function Naming Rules

        i. Allowed characters:
            Letters (a–z, A–Z), digits (0–9), Underscore _, All special characters (@, #, $)

        ii. Must NOT start with a number
            ❌ 1name
            ✅ name1

        iii. No spaces allowed
            ❌ user name
            ✅ userName

        iv. Case-sensitive
            age ≠ Age

        v. Cannot use reserved keywords
            ❌ let, function, return,... so on


    b. Different cases

        i. camelCase

            First word small, next words start with capital letter. No space between words

            Example:
                userName
                totalAmount

            Used for:
                Variables, Functions

        b. PascalCase

            Every word starts with capital letter. No space between words

            Example:
                UserName
                CalculateTotal

            Used for:
                Classes, Constructors, React Components

        c. snake_case

            Words separated by underscore "_" . No space between words

            Example:
                user_name
                total_amount

            Used for:
                Mostly backend, databases

        d. kebab-case

            Words separated by hyphen "-" . No space between words

            Example:
                user-name
                main-header

            Used for:
                File names, CSS classes

        e. Uppercase
        f. Lowercase
